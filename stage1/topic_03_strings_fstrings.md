## Теория

### Строка — неизменяемая последовательность

Строка — объект типа `str`. Её нельзя изменить «на месте»: методы всегда возвращают **новую** строку.

```python
s = "Hello"
s[0] = "h"      # TypeError — нельзя!
s = s.lower()   # OK — создаётся новый объект, s теперь ссылается на него
```

### Создание строк

```python
s1 = 'одинарные кавычки'
s2 = "двойные кавычки"
s3 = """многострочная
строка"""
s4 = r"raw-строка: \n не перевод строки, а буквально \n"  # r-prefix
s5 = b"bytes literal"   # байты, не строка!
```

Raw-строки `r"..."` незаменимы для регулярных выражений (топик 19) и путей к файлам.

### Индексы и срезы

```python
s = "GigabitEthernet0/1"
#    0123456789...       — прямые индексы
#    ...     -3-2-1      — отрицательные (с конца)

s[0]          # "G"
s[-1]         # "1"
s[0:4]        # "Giga"  — stop НЕ включается
s[:4]         # "Giga"  — start=0 по умолчанию
s[4:]         # "bitEthernet0/1"
s[::2]        # каждый второй символ
s[::-1]       # строка наоборот — "1/0terehTabiG..."
len(s)        # 18
```

### Основные методы

```python
s = "  Gi0/1   up/up  \n"

# Очистка
s.strip()         # убрать пробелы/\n с обоих концов
s.lstrip()        # только слева
s.rstrip()        # только справа

# Разбивка и сборка
s.split()         # ["Gi0/1", "up/up"] — по любым пробелам
s.split("/")      # ["  Gi0", "1   up", "up  \n"]
"/".join(["a", "b", "c"])  # "a/b/c"

# Поиск
"Gi" in s                  # True
s.find("Gi")               # индекс первого вхождения или -1
s.index("Gi")              # то же, но выбрасывает ValueError если нет
s.count("0")               # количество вхождений
s.startswith("Gi")         # True/False
s.endswith("up")           # True/False (после strip)

# Замена
s.replace("up", "down")    # заменить все вхождения
s.replace("up", "down", 1) # заменить только первое

# Регистр
s.upper()                  # ВСЁ ЗАГЛАВНЫМИ
s.lower()                  # всё строчными
s.capitalize()             # Первая заглавная
s.title()                  # Каждое Слово Заглавное

# Проверки
"255".isdigit()            # True — все цифры
"Gi0/1".isalnum()          # False — "/" не буква/цифра
"   ".isspace()            # True

# Выравнивание
"r1".ljust(10)             # "r1        " — выровнять влево
"r1".rjust(10)             # "        r1" — выровнять вправо
"r1".center(10)            # "    r1    "
"7".zfill(3)               # "007" — дополнить нулями слева
```

### Конкатенация и эффективность

```python
# Неэффективно для большого числа строк (создаёт новый объект каждый раз):
result = ""
for line in lines:
    result += line + "\n"   # O(n²)

# Правильно — список + один join:
parts = []
for line in lines:
    parts.append(line)
result = "\n".join(parts)   # O(n)
```

### f-строки (f-strings)

Введены в Python 3.6. Самый читаемый способ форматирования:

```python
hostname = "r1"
ip = "10.0.0.1"
uptime = 3.14159

# Базовое использование
f"Device {hostname} at {ip}"

# Форматирование чисел
f"{uptime:.2f}"         # "3.14" — 2 знака после запятой
f"{1024:,}"             # "1,024" — разделитель тысяч
f"{255:#010b}"          # "0b11111111" — двоичное с префиксом
f"{255:#04x}"           # "0xff" — шестнадцатеричное

# Выравнивание
f"{'r1':<10}"           # "r1        " — по левому краю, ширина 10
f"{'r1':>10}"           # "        r1" — по правому
f"{'r1':^10}"           # "    r1    " — по центру
f"{'r1':=<10}"          # "r1========" — заполнитель "="

# Отладка (Python 3.8+)
x = 42
f"{x=}"                 # "x=42" — имя переменной + значение

# Repr для отладки
f"{hostname!r}"         # "'r1'" — как repr()
f"{hostname!s}"         # "r1"  — как str() (по умолчанию)

# Выражения внутри
f"{'up' if status else 'down'}"
f"{', '.join(['Gi0/1', 'Gi0/2'])}"
```

### Другие способы форматирования

```python
# str.format() — старый, но встречается в коде
"Device {} at {}".format(hostname, ip)
"Device {host} at {ip}".format(host="r1", ip="10.0.0.1")

# % — очень старый стиль, только для чтения
"Device %s at %s" % (hostname, ip)
```

## Примеры (сетевые)

```python
# Парсинг строки вывода CLI
line = "GigabitEthernet0/0      192.168.1.1     YES manual up                    up"

# Нормализация названия интерфейса

# Генерация команды
for vlan_id, vlan_name in [(10, "USERS"), (20, "SERVERS")]:
    print(f"vlan {vlan_id}")
    print(f" name {vlan_name}")

# Выровненная таблица
devices = [("r1", "10.0.0.1", "up"), ("sw1", "10.0.0.2", "down")]
print(f"{'Hostname':<12} {'IP':<16} {'Status'}")
print("-" * 36)
for host, ip, st in devices:
    print(f"{host:<12} {ip:<16} {st}")
```

## Частые ошибки

```python
# Методы строк возвращают новую строку — присвоение обязательно!
s = "  hello  "
s.strip()           # ничего не изменилось!
s = s.strip()       # правильно

# Осторожно с split("/") — разные результаты
"a/b/c".split("/")     # ["a", "b", "c"]
"a/b/c".split("/", 1)  # ["a", "b/c"] — максимум 1 разбивка

# join — разделитель стоит СЛЕВА от метода
", ".join(["a", "b"])   # "a, b"
["a", "b"].join(", ")   # AttributeError — список, не строка!
```
