## Цели

- Применять множества для быстрых операций с уникальными коллекциями (VLAN, порты, IP).

## Теория

### Кортеж (tuple)

Кортеж — **неизменяемая** упорядоченная последовательность. Создаётся один раз, изменить нельзя.

```python
# Создание
empty = ()
single = (42,)       # ВАЖНО: запятая обязательна для кортежа из 1 элемента!
single_wrong = (42)  # это просто int 42, не кортеж!

coords = (48.8566, 2.3522)      # широта, долгота
iface_info = ("Gi0/1", "up", 1000)

# Доступ — как у списка, но без изменения
coords[0]       # 48.8566
coords[-1]      # 2.3522
coords[0:]      # (48.8566, 2.3522) — срез возвращает кортеж

# Нельзя:
coords[0] = 0.0  # TypeError
```

#### Зачем нужен кортеж если есть список?

1. **Неизменяемость** — гарантия, что данные не изменятся случайно.
2. **Ключ словаря** — кортежи хешируемы, списки — нет.
3. **Чуть быстрее** списка при итерации.
4. **Семантика** — кортеж обычно означает «запись фиксированной структуры».

```python
# Кортеж как ключ словаря
arp_table = {
    ("10.0.0.1", "Gi0/0"): "aabb.cc00.6500",
    ("10.0.0.2", "Gi0/1"): "aabb.cc00.6510",
}
arp_table[("10.0.0.1", "Gi0/0")]  # "aabb.cc00.6500"
```

#### Распаковка кортежей

```python
# Базовая распаковка
ip, mask = ("192.168.1.0", "255.255.255.0")
host, port = "r1", 22

# Расширенная распаковка
first, *rest = (1, 2, 3, 4, 5)
# first = 1, rest = [2, 3, 4, 5]

first, *middle, last = (1, 2, 3, 4, 5)
# first = 1, middle = [2, 3, 4], last = 5

# Игнорирование ненужных значений
_, port, _ = ("tcp", 22, "ssh")  # нас интересует только port
```

#### Возврат нескольких значений из функции

Технически это кортеж — распаковывается автоматически:

```python
def parse_interface(line: str) -> tuple[str, str, str]:
    parts = line.split()
    return parts[0], parts[4], parts[5]   # возвращает кортеж

iface, status, protocol = parse_interface("Gi0/1  10.0.0.1  YES  manual  up  up")
```

#### Именованные кортежи (namedtuple / dataclass)

```python
from collections import namedtuple

Device = namedtuple("Device", ["hostname", "ip", "role"])
r1 = Device("r1", "10.0.0.1", "router")

r1.hostname   # "r1" — доступ по имени
r1[0]         # "r1" — доступ по индексу
r1._asdict()  # {"hostname": "r1", "ip": "10.0.0.1", "role": "router"}
```

### Множество (set)

Множество — **неупорядоченная** коллекция **уникальных** хешируемых элементов. Быстрая проверка принадлежности O(1).

```python
# Создание
empty = set()         # НЕ {} — это пустой словарь!
vlans = {10, 20, 30}
from_list = set([10, 20, 20, 30, 30])  # {10, 20, 30} — дубликаты убраны

# Добавление и удаление
vlans.add(40)         # добавить один элемент
vlans.update([50, 60]) # добавить несколько
vlans.discard(99)     # удалить если есть (без ошибки)
vlans.remove(10)      # удалить (KeyError если нет)
vlans.pop()           # удалить произвольный элемент

# Принадлежность — O(1), намного быстрее list!
10 in vlans           # True
99 not in vlans       # True
```

#### Операции над множествами

```python
a = {10, 20, 30, 40}
b = {20, 40, 60}

# Объединение — элементы из обоих
a | b           # {10, 20, 30, 40, 60}
a.union(b)      # то же

# Пересечение — только общие элементы
a & b           # {20, 40}
a.intersection(b)

# Разность — есть в a, нет в b
a - b           # {10, 30}
a.difference(b)

# Симметричная разность — есть в одном, но не в обоих
a ^ b           # {10, 30, 60}
a.symmetric_difference(b)

# Проверки
{20, 40}.issubset(a)    # True — {20,40} ⊆ a
a.issuperset({20, 40})  # True
a.isdisjoint({99, 100}) # True — нет общих элементов
```

#### Frozenset — неизменяемое множество

```python
fs = frozenset({10, 20, 30})
fs.add(40)  # AttributeError — нельзя изменить

# Может быть ключом словаря или элементом другого множества
allowed_vlans = frozenset({10, 20, 30})
policy = {allowed_vlans: "permit"}
```

#### Set comprehension

```python
# Уникальные первые октеты из списка IP
ips = ["10.0.0.1", "10.0.0.2", "192.168.1.1", "192.168.1.2"]
first_octets = {ip.split(".")[0] for ip in ips}
# {"10", "192"}
```

## Примеры (сетевые)

```python
# Сравнение конфигурированных и активных VLAN
configured = {10, 20, 30, 40}
active      = {10, 30}

missing   = configured - active    # {20, 40} — настроены, но неактивны
extra     = active - configured    # {} — активны, но не настроены
common    = configured & active    # {10, 30}

print(f"Не подняты: {sorted(missing)}")
print(f"Лишние: {sorted(extra)}")

# Быстрая проверка допустимости порта
MGMT_PORTS = frozenset({22, 80, 443, 830, 8443})
for port in [22, 8080, 443]:
    status = "allowed" if port in MGMT_PORTS else "denied"
    print(f"Port {port}: {status}")

# Уникальные MAC из таблицы
mac_list = ["aabb.cc00.6500", "dead.beef.0001", "aabb.cc00.6500"]
unique_macs = set(mac_list)
print(f"Уникальных MAC: {len(unique_macs)}")  # 2
```

## Частые ошибки

```python
# Пустое множество — только set(), НЕ {}
empty_set  = set()   # правильно
empty_dict = {}      # это словарь!

# Кортеж из одного элемента требует запятую
t = (42,)    # tuple
t = (42)     # int — НЕ кортеж!

# Порядок в set не гарантирован
s = {3, 1, 2}
list(s)   # может быть [1, 2, 3] или [3, 1, 2] — не предсказуемо
sorted(s) # [1, 2, 3] — для отчёта всегда сортируйте

# set содержит только хешируемые объекты
{[1, 2]}  # TypeError — список не хешируем
{(1, 2)}  # OK — кортеж хешируем
```
