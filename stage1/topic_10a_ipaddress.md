# Топик 10a. Модуль `ipaddress`

## Цели

- Работать с IPv4/IPv6-адресами и сетями без ручного разбора строк.
- Итерировать хосты, проверять принадлежность, сравнивать и сортировать адреса.

## Теория (тезисы)

- `ipaddress` — часть **стандартной библиотеки** Python (3.3+), ничего устанавливать не нужно.
- **`ip_address(str)`** — объект `IPv4Address` или `IPv6Address`; сравниваются, сортируются.
- **`ip_network(str)`** — объект сети; `strict=False` игнорирует биты хоста (`192.168.1.5/24` → `/24`).
- **`ip_interface(str)`** — адрес + маска вместе; даёт и `.ip`, и `.network`.
- Полезные атрибуты сети: `.network_address`, `.broadcast_address`, `.prefixlen`, `.num_addresses`, `.hosts()`.
- Проверка принадлежности: `addr in network` — стандартный оператор `in`.

## Примеры

```python
import ipaddress

net = ipaddress.ip_network("10.0.0.0/24")
print(list(net.hosts())[:3])  # [IPv4Address('10.0.0.1'), ...]

addr = ipaddress.ip_address("10.0.0.50")
print(addr in net)  # True

iface = ipaddress.ip_interface("192.168.1.10/30")
print(iface.network)  # IPv4Network('192.168.1.8/30')
print(iface.network.broadcast_address)  # IPv4Address('192.168.1.11')
```

## Практика

1. Из строки `"10.1.2.3/22"` получите: сетевой адрес, broadcast, количество хостов и первые 5 IP хостов.
2. Напишите функцию `is_rfc1918(ip_str: str) -> bool`, возвращающую `True`, если IP входит в одну из трёх частных сетей RFC 1918 (`10/8`, `172.16/12`, `192.168/16`).
3. Из списка строк `["192.168.0.5", "10.0.0.1", "172.16.0.1", "8.8.8.8"]` отделите публичные адреса от частных.
4. **Сеть:** дан список адресов из разных VLAN (`10.10.1.x/24`, `10.10.2.x/24`, …); сгруппируйте их по принадлежащей сети (ключ — строка сети, значение — список IP).

## Самопроверка

- Не парсите IP вручную через `split(".")` — для этого есть `ipaddress`.
- `ip_address` выбросит `ValueError` при неверном вводе — ловите в том же блоке `try`, что другие сетевые ошибки.
