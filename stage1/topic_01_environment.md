# Топик 1. Окружение и первый скрипт

## Цели

- Установить Python 3 и понимать, что происходит при запуске скрипта.
- Уметь создавать и активировать виртуальное окружение.
- Знать базовые команды REPL и структуру типичного Python-проекта.

---

## Теория

### Как Python выполняет код

Python — **интерпретируемый** язык: интерпретатор читает файл `.py` построчно и выполняет каждую строку сразу. Никакой предварительной компиляции в машинный код нет (хотя Python компилирует в байткод `.pyc` для ускорения повторного запуска — папка `__pycache__`).

```
my_script.py  →  интерпретатор python3  →  результат
```

Файл `.py` — это обычный текст в кодировке **UTF-8**. Открыть и редактировать можно в любом редакторе.

### Версии Python

Всегда используйте **Python 3** (3.11+). На macOS может быть предустановлен системный Python 2 (`python`) — никогда не используйте его для своих проектов. Ваш интерпретатор: `python3`.

```bash
python3 --version    # убедитесь, что 3.11 или новее
which python3        # путь к интерпретатору
```

### Виртуальное окружение (venv)

Каждый проект должен иметь **изолированное** окружение — чтобы библиотека версии X для одного проекта не конфликтовала с версией Y для другого.

```bash
# Создать окружение в папке .venv
python3 -m venv .venv

# Активировать
source .venv/bin/activate

# Убедиться — в начале строки появится (.venv)
which python3        # должен указывать на .venv/bin/python3

# Деактивировать когда закончили
deactivate
```

После активации `python3`, `pip` и все установленные пакеты берутся из `.venv`, а не из системы.

### pip — установка пакетов

```bash
pip install netmiko            # установить последнюю версию
pip install netmiko==4.3.0     # конкретная версия
pip install -r requirements.txt  # всё из файла зависимостей
pip list                       # что установлено
pip freeze > requirements.txt  # зафиксировать текущие версии
```

### REPL — интерактивный режим

```bash
python3           # запустить REPL
>>> 2 ** 10       # выражение вычисляется и выводится
1024
>>> exit()        # выход
```

REPL полезен для быстрых экспериментов. Для реального кода — всегда файл `.py`.

### Запуск скрипта

```bash
python3 my_script.py
python3 -c "print('hello')"   # однострочник прямо в терминале
python3 -m module_name        # запуск модуля из стандартной библиотеки
```

### Структура проекта

```
my_project/
├── .venv/              # виртуальное окружение — НЕ в git
├── .gitignore          # список игнорируемых файлов
├── requirements.txt    # зависимости проекта
├── main.py             # точка входа
└── utils.py            # вспомогательные функции
```

Минимальный `.gitignore` для Python:
```
.venv/
__pycache__/
*.pyc
.env
```

### PEP 8 — стиль кода

PEP 8 — официальное руководство по стилю Python. Ключевые правила:

| Правило | Пример |
|---------|--------|
| Отступы — 4 пробела (не таб) | `    return x` |
| Имена функций и переменных — `snake_case` | `get_hostname` |
| Имена классов — `PascalCase` | `NetworkDevice` |
| Константы — `UPPER_CASE` | `DEFAULT_TIMEOUT = 30` |
| Пустые строки: 2 между функциями, 1 внутри | — |
| Длина строки — до 88–100 символов | — |

### print() подробнее

```python
print("a", "b", "c")           # a b c  (sep по умолчанию " ")
print("a", "b", sep="-")       # a-b
print("Hello", end="")         # без перевода строки
print(f"IP: {ip}, port: {port}")  # f-строка (подробно в топике 3)
```

## Примеры

```python
# hello.py — минимальный скрипт
hostname = "r1"
print(f"Device: {hostname}")
```

```bash
# Полный цикл создания проекта
mkdir lab_project && cd lab_project
python3 -m venv .venv
source .venv/bin/activate
echo "print('Hello, network!')" > hello.py
python3 hello.py
```

## Практика

1. Создайте каталог `lab_project/`, создайте и активируйте `.venv`, проверьте `which python3` — путь должен указывать внутрь `.venv`.
2. Напишите `hello.py`: задайте переменную `hostname = "r1"` и выведите `"Device: r1 — ready"` через f-строку.
3. Создайте файл `requirements.txt` с одной строкой `# пока пусто` и попробуйте `pip install -r requirements.txt` — убедитесь, что ошибок нет.
4. Запустите REPL, вычислите `2 ** 32` (максимум IPv4-адресов), `255 & 192` (операция AND масок) и выйдите.
5. Создайте `.gitignore` для проекта со строками `.venv/`, `__pycache__/`, `*.pyc`.
6. **Codewars-стиль:** напишите функцию `greet(name: str) -> str`, возвращающую `"Hello, <name>!"` (без лишних пробелов). Протестируйте в REPL.

## Частые ошибки

- Запуск `python` вместо `python3` — может попасть на Python 2.
- Установка пакетов без активированного venv — они попадут в системный Python.
- Редактирование файлов внутри `.venv` — это директория окружения, не проекта.
- Добавление `.venv` в git — тысячи лишних файлов.

---

## Самопроверка

- `python3 hello.py` запускается без ошибок из каталога проекта.
- `pip list` показывает пакеты только вашего venv, а не системы.
- Папки `.venv` нет в git-репозитории.
