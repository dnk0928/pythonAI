## Нейросети и ассистенты (2026): как не потерять эффективность

Используйте ИИ **как ускоритель**, а не замену понимания.

| Делать | Не делать |
|--------|-----------|
| Просить объяснить traceback и альтернативы кода | Вставлять код на собеседовании без понимания |
| Генерировать **тесты** к своим функциям и проверять их глазами | Сдавать «как есть» код с секретами и `except: pass` |
| Разбирать сложный вывод `show` / XML с подсказками | Просить решить за вас всю задачу без попытки самому |
| Черновик TextFSM/Jinja2, затем правка вручную | Доверять «идеальному» regex без прогона на реальных логах |

**Правило:** если не можете объяснить решение вслух за 2 минуты — вернитесь к топику или к Core Python.

## Быстрая оценка «на каком я этапе по навыкам»

Выберите один вариант **по каждой строке** (отметьте букву).

1. **Python Core:** A — только синтаксис читаю · B — пишу скрипты до ~100 строк · C — модули, пакеты, свой мини-проект без помощи.
2. **Сеть:** A — не подключался по SSH из кода · B — Netmiko к одному хосту · C — массово + инвентарь + ошибки под контролем.
3. **Данные:** A — только print · B — JSON/YAML руками · C — схемы + парсинг + отчёт.
4. **[2026]** A — ничего из Stage 4 · B — попробовал 1–2 темы · C — async + REST API + pydantic в проекте.

**Сводка:**
- Везде **A** → начало этапа 1; сначала топики 1–5.
- **B** по всем → середина курса; фокус на этапе 2 и регулярных заданиях.
- **C** по 1–3 → выпускной проект + Stage 4 бонус.
- **C** везде → готов к техническому собеседованию на позицию network automation engineer.
