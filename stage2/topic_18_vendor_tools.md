# Топик 18. Вендор-инструменты: pyATS (Cisco) и PyEZ (Juniper)

## Цели

- Понимать **назначение** pyATS/Genie и Junos PyEZ без прохождения полного курса по продукту.
- Знать, когда выбрать вендор-инструмент, а когда — общий стек (Netmiko/NAPALM).

## Теория (тезисы)

### pyATS / Genie (Cisco)

- **pyATS** — фреймворк тестирования и автоматизации сети Cisco: топология, повторяемые запуски, отчёты.
- **Genie** парсер — библиотека **готовых парсеров** `show` команд → структурированный JSON (сильная сторона при сложном CLI).
- Типичный сценарий: снять «снимок» сети до/после изменения, сравнить.

### Junos PyEZ

- **PyEZ** — высокоуровневый API к Junos: факты устройства, RPC, конфигурация через `jnpr.junos`.
- Удобно, если ваша среда **в основном Juniper**.

## Практика (обзорные, без обязательного железа)

1. **Cisco:** установите `pyats[full]` в отдельное venv (тяжёлый мета-пакет); выполните `genie parse "show version" --os iosxe --output /tmp/out.json` на сохранённом выводе из лаба (см. документацию Genie CLI).
2. **Juniper:** прочитайте раздел «Device facts» в PyEZ и запишите псевдокод, как бы вы получили `facts` с устройства.
3. Составьте таблицу для себя: **критерий** → **Netmiko / NAPALM / pyATS / PyEZ**.
