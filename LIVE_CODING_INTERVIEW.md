# Live-coding и собеседование (раздел курса)

Цель — тренировать **короткие задачи** на **ядре Python** и уметь **проговаривать ход мыслей**, как на интервью.

## Как заниматься

1. Поставьте таймер **15–20 минут** на задачу.
2. Решайте **без IDE-подсказок** сначала, затем сравните с эталоном в документации.
3. Вслух формулируйте: вход → граничные случаи → структуры данных → сложность по времени (грубо).

## Задачи уровня Codewars (8–7 kyu)

Подберите на [Codewars](https://www.codewars.com/) по тегам **Fundamentals**, **Strings**, **Arrays**:

1. Подсчёт символов / слов в строке с фильтрацией.
2. Разворот строки по словам, сохраняя пробелы (упрощённые варианты).
3. Найти первый неповторяющийся символ / индекс.
4. Сумма чисел в строке с разделителями.
5. Проверка скобочной последовательности.
6. Преобразование snake_case ↔ PascalCase.
7. FizzBuzz с правилами на делимость (классика интервью).

## Сетевые мини-задачи (без железа)

Решите на чистом Python, используя только строки и структуры данных:

1. Разобрать строку маршрута `ip route`: сеть, маска/prefix, next-hop (упрощённый формат).
2. Валидировать октеты IPv4 до сборки адреса (без `ipaddress` на первом проходе).
3. Нормализовать MAC к формату `aa:bb:cc:dd:ee:ff` из `aaaa.bbbb.cccc`.

## Чеклист «как объяснять»

- Назвать **ограничения** входа (пустая строка, регистр, лишние пробелы).
- Сказать, почему выбран **list vs dict**.
- Упомянуть **тесты**: 1–2 граничных случая до написания кода.
- Если застряли — озвучить план B (например, regex vs `split`).
