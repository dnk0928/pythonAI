# Дополнения к курсу: эффективность и актуальность (2026)

Краткий список того, **что имеет смысл усилить** поверх уже существующих топиков — без перегруза теорией. Используйте как дорожную карту самостоятельно или для доработки материалов.

## Уже сильные стороны курса

- Чёткое разделение на Core → сеть → парсинг/шаблоны/проекты.
- Практика в каждом топике и офлайн-данные в `labs/`.
- Выпускной проект с критериями и блоком live-coding.
- Stage 4 с современным стеком 2026 года.

## Что улучшить для большей эффективности

### 1. Ритм и проверка усвоения

- После **каждых 3 топиков** — короткая самопроверка: 30 минут без документации на задачу из пройденного (например, разбор файла из `labs/`).
- Вести **один** репозиторий с решениями и датами в [PROGRESS.md](PROGRESS.md).

### 2. Качество кода (современный минимум 2026)

- Зафиксировать версию Python (**3.11+**): удобнее отладка и сообщения об ошибках.
- Добавить к финальным проектам: **`ruff check`** или `flake8` и форматирование `black` — как опцию «уровень pro».
- Один файл тестов **`pytest`** на функцию парсинга или нормализации MAC — резко снижает регрессии. Подробнее: [stage4/topic_b6_pytest_network.md](stage4/topic_b6_pytest_network.md).

### 3. Сеть и масштаб

- После Netmiko/Scrapli — явно проговорить: **когда нужен Nornir/async** (сотни устройств, таймауты). Ответ в [stage4/topic_b1_asyncio_scrapli.md](stage4/topic_b1_asyncio_scrapli.md).
- Короткая вставка: **REST/HTTP** (`httpx`) для контроллеров и облаков — [stage4/topic_b3_httpx_rest_api.md](stage4/topic_b3_httpx_rest_api.md).

### 4. Безопасность и гигиена (обязательно в 2026)

- Никогда не коммитить пароли; **`.env` + `python-dotenv`** или только переменные окружения — упомянуть в README курса и в выпускном проекте.
- Для ИИ-инструментов: не вставлять **реальные** IP/пароли/выводы прод-сети в публичные чаты. Подробнее: [stage4/topic_b5_ai_llm_netops.md](stage4/topic_b5_ai_llm_netops.md).

### 5. Нейросети в обучении (2026-специфика)

- **Плюс:** объяснение ошибок, варианты рефакторинга, генерация тест-кейсов, шаблон README.
- **Минус:** замена изучения Core Python — на собеседовании это быстро вскрывается.
- Рекомендуемое правило: **сначала** решение своими силами или половина задачи, **потом** сравнение с ответом ассистента.

**Инструменты ИИ для сетевого инженера 2026:**

| Инструмент | Применение |
|------------|------------|
| **Cursor / GitHub Copilot** | IDE-подсказки, рефакторинг, объяснение кода |
| **ChatGPT / Claude** (веб) | Объяснение traceback, черновик шаблона, обзор документации |
| **OpenAI API** (скрипт) | Автоматизированный анализ логов, генерация конфига — [B5](stage4/topic_b5_ai_llm_netops.md) |
| **litellm** | Единый клиент для замены провайдера без рефакторинга |
| **ollama** (локально) | Конфиденциальные данные: production-конфиги, реальные IP |
