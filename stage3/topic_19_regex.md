# Топик 19. Регулярные выражения (`re`)

## Цели

- Искать и извлекать подстроки по шаблону; использовать группы.
- Не злоупотреблять regex там, где достаточно `split` или TextFSM.

## Теория (тезисы)

- **`re.search`** — первое совпадение; **`re.findall`** — все совпадения; **`re.sub`** — замена.
- **Сырые строки** `r"..."` упрощают экранирование обратных слешей.
- **Группы** `(...)` — извлечь часть совпадения; именованные `(?P<name>...)`.
- **Жадность** квантификаторов: `*?` + non-greedy при необходимости.
- Для сложного CLI предпочтительны **TextFSM** или **vendor parsers** — regex как вспомогательный инструмент.

## Примеры

```python
import re

m = re.search(r"(\d+\.\d+\.\d+\.\d+)", line)
if m:
    ip = m.group(1)
```

## Практика

1. Из файла `show_version_ios.txt` извлеките версию IOS вида `Version 15.7(3)M8` целиком.
2. Из строки MAC `aaaa.bbbb.cccc` получите канонический `aa:aa:bb:bb:cc:cc` (или другой формат по шаблону).
3. Найдите все IPv4 в тексте лога (упрощённый паттерн достаточен для учебы).
4. **Сеть:** из вывода `show ip int brief` (файл в лабе) извлеките пары `(interface, ip)` для строк со статусом `up`.

## Самопроверка

- Компилируйте часто используемые паттерны: `PHONE_RE = re.compile(r"...")`.
