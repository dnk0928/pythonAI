# Топик 26. Мини-проект B — шаблон и выкат (lab)

## Цель

Сгенерировать конфигурацию через **Jinja2** и применить её на **группе lab-устройств** через **Netmiko** (`send_config_set`).

## Постановка

1. Возьмите инвентарь и добавьте переменные: например, описание интерфейса `Gi0/1` или список SNMP community (только для lab).
2. Напишите шаблон `templates/lab_banner.j2` с баннером `motd`, содержащим `hostname` и дату генерации.
3. Скрипт `deploy_banner.py` рендерит конфиг для каждого хоста и вызывает `send_config_set` в режиме merge.
4. После выката выполните проверочную команду `show running-config | include banner` (или аналог) и сохраните вывод.

## Безопасность

- Только **тестовая** среда; логин/пароль из переменных окружения.
- Сделайте **rollback-план**: сохраните `show run` до изменения.

## Дополнительно

- Оберните выкат в **Nornir** (если уже прошли топик 14).

## Критерии приёмки

- Один шаблон — много устройств; различия только в контексте.
- Ошибки применения на одном хосте не роняют весь прогон (агрегируйте в отчёт).
