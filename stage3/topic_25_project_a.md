# Топик 25. Мини-проект A — отчёт по uptime (CSV)

## Цель

Скомбинировать **Netmiko**, **файлы**, **исключения** и инвентарь для массового сбора «времени работы» устройств.

## Постановка

1. Прочитайте список устройств из [`../labs/inventory.json`](../labs/inventory.json) (или YAML).
2. Для каждого устройства подключитесь по **Netmiko** и выполните команду, возвращающую uptime (например `show version` на IOS или `show system uptime` — подберите под вашу платформу).
3. Из вывода извлеките **строку uptime** (regex или простой поиск подстроки на первом этапе).
4. Запишите результат в CSV `artifacts/uptime_report.csv` с колонками: `hostname,host,ip,uptime_raw,ok,error`.
5. Если подключение не удалось — строка с `ok=false` и текстом ошибки в `error`.

## Офлайн-режим

- Флаг `USE_MOCK=1` или переменная окружения: читайте готовые файлы из `labs/sample_outputs/` по имени хоста вместо SSH.

## Критерии приёмки

- Повторный запуск перезаписывает отчёт детерминированно.
- Нет необработанных исключений наружу.
- Код разбит на функции: `load_inventory`, `fetch_uptime`, `write_csv`, `main`.

## Подсказки

- Используйте `csv.DictWriter` из стандартной библиотеки.
- Логируйте время выполнения в конце (опционально).
